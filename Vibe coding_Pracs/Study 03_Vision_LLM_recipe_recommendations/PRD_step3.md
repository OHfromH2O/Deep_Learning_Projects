# PRD Step 3 — 사용자 프로필 & 레시피 저장

## 개요
사용자 계정을 생성하고, Step 2에서 마음에 든 레시피를 저장·관리할 수 있는 개인 레시피 보관함을 제공한다.

---

## 목표
- 회원가입/로그인 없이도 앱을 사용할 수 있되, 저장 기능은 로그인 후에만 활성화한다.
- 저장된 레시피를 나중에 다시 조회하고 관리할 수 있다.
- 사용자 선호 정보(식이 제한 등)를 프로필에 저장하여 레시피 생성 시 반영한다.

---

## 사용 기술
| 항목 | 내용 |
|------|------|
| 인증 | 브라우저 `localStorage` 기반 간단 프로필 (외부 인증 서버 없음) |
| 데이터 저장 | `localStorage` (JSON 직렬화) |
| 프론트엔드 | HTML / CSS / JavaScript (Vanilla) |

> **Note:** 실서비스 전환 시 Firebase Auth + Firestore 등으로 교체를 권장한다.

---

## 기능 요구사항

### FR-1. 프로필 생성 및 관리
- 최초 진입 시 닉네임 입력만으로 프로필을 생성한다 (이메일·비밀번호 불필요).
- 프로필 편집 화면에서 수정 가능한 항목:
  - 닉네임
  - 프로필 이미지 (로컬 파일 업로드)
  - 식이 제한: 채식, 비건, 글루텐프리, 유제품 없음 (다중 선택)
  - 기피 재료 목록 (텍스트 태그로 입력)
- 저장된 식이 제한과 기피 재료는 Step 2 레시피 생성 프롬프트에 자동으로 포함된다.

### FR-2. 레시피 저장
- Step 2의 "저장하기" 버튼 클릭 시 해당 레시피를 프로필의 저장 목록에 추가한다.
- 저장 시 함께 기록되는 메타데이터:
  - 저장 날짜
  - 인식에 사용된 재료 목록 (Step 1 결과)
  - 생성에 사용된 모델 이름
- 이미 저장된 레시피를 다시 저장하려 하면 중복 저장 여부를 확인하는 안내를 표시한다.

### FR-3. 레시피 보관함
- 저장된 레시피를 카드 목록으로 표시한다.
- 정렬 옵션: 저장 날짜순 (기본), 요리 이름순
- 각 카드에서 가능한 동작:
  - 상세 내용 펼쳐보기 (Step 2와 동일한 레이아웃)
  - 메모 추가 (자유 텍스트)
  - 삭제
- 저장된 레시피가 없을 때는 빈 상태 안내 문구와 함께 "레시피 만들러 가기" 버튼을 표시한다.

### FR-4. 프로필 초기화
- 설정 화면에서 "데이터 초기화" 버튼으로 저장된 레시피와 프로필을 모두 삭제할 수 있다.
- 삭제 전 확인 다이얼로그를 표시한다.

---

## 비기능 요구사항
- `localStorage` 용량 한계(약 5MB)에 근접하면 오래된 레시피 삭제를 권장하는 안내를 표시한다.
- 앱 새로고침 후에도 프로필과 저장 레시피가 유지되어야 한다.
- 식이 제한·기피 재료 정보가 레시피 생성 프롬프트에 반영되는지 Step 2와의 연동을 명확히 정의한다.

---

## Step 2 프롬프트 연동 예시
사용자가 "채식", 기피 재료 "새우"를 설정한 경우, Step 2 프롬프트에 아래 내용이 추가된다:
> "단, 다음 조건을 반드시 지켜주세요: 채식 식단 (육류·해산물 제외), 새우는 사용하지 마세요."
