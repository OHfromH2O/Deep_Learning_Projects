# PRD Step 1 — 냉장고 이미지 인식

## 개요
사용자가 냉장고 사진을 업로드하면, Vision 모델이 이미지를 분석하여 식재료 목록을 추출한다.

---

## 목표
- 사용자가 냉장고 사진 한 장을 업로드하는 것만으로 재료 목록을 자동 생성한다.
- 추출된 재료 목록은 Step 2(레시피 생성)의 입력으로 전달된다.

---

## 사용 기술
| 항목 | 내용 |
|------|------|
| AI 모델 | `nvidia/nemotron-nano-12b-v2-vl:free` (Vision-Language) |
| API | OpenRouter `/v1/chat/completions` |
| 이미지 전송 방식 | 클라이언트에서 base64 인코딩 후 data URI로 전송 |
| 프론트엔드 | HTML / CSS / JavaScript (Vanilla) |

---

## 기능 요구사항

### FR-1. 이미지 업로드
- 파일 선택(input[type=file]) 및 드래그&드롭 두 가지 방식을 지원한다.
- 허용 포맷: JPEG, PNG, WEBP
- 최대 파일 크기: 10MB
- 업로드 즉시 미리보기 이미지를 화면에 표시한다.

### FR-2. 재료 인식 요청
- 업로드된 이미지를 base64로 인코딩하여 OpenRouter API에 전송한다.
- 전송 시 사용할 프롬프트:
  > "이 냉장고 사진에서 보이는 식재료를 모두 파악하여, JSON 배열 형식으로만 반환해 주세요.
  > 형식 예시: [\"달걀\", \"우유\", \"당근\", \"버터\"]
  > 식재료가 아닌 설명은 포함하지 마세요."
- API 호출 중에는 로딩 스피너를 표시한다.

### FR-3. 결과 표시
- 인식된 재료를 태그(chip) 형태로 화면에 나열한다.
- 사용자가 개별 태그를 삭제하거나 새 재료를 직접 추가할 수 있다.
- "레시피 추천받기" 버튼을 누르면 확정된 재료 목록을 Step 2로 전달한다.

---

## 비기능 요구사항
- API 응답 시간이 15초를 초과하면 타임아웃 안내 메시지를 표시한다.
- API가 JSON 배열 외의 형식을 반환할 경우, 텍스트를 파싱하여 재료를 추출하는 폴백 로직을 포함한다.
- 모바일 화면(최소 375px)에서도 동작해야 한다.
