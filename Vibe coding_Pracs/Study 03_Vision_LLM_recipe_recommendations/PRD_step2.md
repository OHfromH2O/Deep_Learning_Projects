# PRD Step 2 — 레시피 생성

## 개요
Step 1에서 추출한 재료 목록을 바탕으로 LLM이 실현 가능한 레시피를 생성하고, 사용자에게 카드 형태로 제시한다.

---

## 목표
- 보유 재료만으로 만들 수 있는 레시피 3가지를 생성한다.
- 각 레시피는 재료·조리 순서·소요 시간을 포함한 구조화된 형식으로 제공한다.
- 사용자가 레시피를 Step 3(저장)으로 넘길 수 있다.

---

## 사용 기술
| 항목 | 내용 |
|------|------|
| AI 모델 | `nvidia/nemotron-3-super-120b-a12b:free` (Text LLM) |
| API | OpenRouter `/v1/chat/completions` |
| 입력 | Step 1에서 전달된 재료 배열 (예: `["달걀", "우유", "버터"]`) |
| 출력 형식 | JSON 구조체 배열 |

---

## 기능 요구사항

### FR-1. 레시피 생성 요청
- Step 1에서 확정된 재료 목록을 받아 아래 프롬프트로 API를 호출한다.
- 사용할 프롬프트:
  > "다음 재료들로 만들 수 있는 요리 레시피 3가지를 JSON 배열로 반환해 주세요.
  > 재료 목록: {ingredients}
  > 각 레시피의 형식:
  > {
  >   \"name\": \"요리 이름\",
  >   \"description\": \"한 줄 설명\",
  >   \"cooking_time\": \"소요 시간 (예: 20분)\",
  >   \"difficulty\": \"난이도 (쉬움/보통/어려움)\",
  >   \"ingredients\": [\"재료1 적정량\", \"재료2 적정량\"],
  >   \"steps\": [\"1단계 설명\", \"2단계 설명\"]
  > }
  > JSON 배열 외의 텍스트는 포함하지 마세요."
- 호출 중 로딩 애니메이션을 표시한다.

### FR-2. 레시피 카드 표시
- 생성된 레시피 3개를 카드 UI로 나란히 표시한다.
- 각 카드에 포함되는 정보:
  - 요리 이름, 한 줄 설명
  - 소요 시간 및 난이도 뱃지
  - 필요 재료 목록
  - 단계별 조리 방법 (순서형 목록)
- 카드를 클릭하면 모달 또는 상세 패널에서 전체 내용을 확인할 수 있다.

### FR-3. 재시도 및 재료 수정
- "다시 추천받기" 버튼으로 동일 재료로 새 레시피를 재생성할 수 있다.
- "재료 수정하기" 버튼으로 Step 1 화면으로 돌아갈 수 있다.

### FR-4. 레시피 저장 연결
- 각 레시피 카드에 "저장하기" 버튼을 제공한다.
- 버튼 클릭 시 Step 3(사용자 프로필 & 저장)으로 해당 레시피 데이터를 전달한다.
- 비로그인 상태에서 저장 시도 시 로그인/회원가입 안내 메시지를 표시한다.

---

## 비기능 요구사항
- API 응답 시간이 30초를 초과하면 타임아웃 안내 메시지를 표시한다.
- JSON 파싱 실패 시 raw 텍스트를 파싱하는 폴백 로직을 포함한다.
- 재료 수가 2개 미만일 경우 사용자에게 재료 추가를 유도하는 안내 문구를 표시한다.
